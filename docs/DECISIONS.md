## 决策列表

### DEC-001: 保留 DataStore 抽象层

- **日期**: 2026-02-18
- **状态**: ✅ 已决策
- **提出者**: 小F（前端开发）
- **评估者**: 小M（架构师）
- **决策**: 在 Nuxt 3 中保留现有 DataStore 接口抽象，前端实现 ApiDataStore（HTTP 调用），后端实现 DbDataStore（Drizzle + SQLite）
- **理由**: 上层 Service 代码改动最小化，迁移风险低
- **影响**: HabitService 等业务层几乎不需要重写

### DEC-002: 服务端数据库选型 SQLite + Drizzle ORM

- **日期**: 2026-02-18
- **状态**: ✅ 已决策
- **提出者**: 小I（后端开发）
- **评估者**: 小M（架构师）
- **决策**: 采用 SQLite + Drizzle ORM
- **理由**: 个人助手单用户场景，SQLite 零配置零依赖；Drizzle TypeScript-first 类型安全，轻量，支持未来迁移到 PostgreSQL
- **否决方案**: Prisma（太重）、better-sqlite3 裸用（无类型安全）、TypeORM（维护不积极）
- **影响**: 数据文件即数据库，部署简单

### DEC-003: 图表方案使用纯 SVG

- **日期**: 2026-02-18
- **状态**: ✅ 已决策
- **提出者**: 小Y（产品经理）
- **决策**: 习惯历史图表使用纯 SVG 实现，不引入外部图表库
- **理由**: 保持项目轻量，黑白色调设计自定义更灵活
- **影响**: 年度热力图 + 月度趋势折线图均用 SVG 手绘

### DEC-004: 数据同步策略采用乐观更新 + refetch

- **日期**: 2026-02-18
- **状态**: ✅ 已决策
- **提出者**: 小M（架构师）
- **决策**: 原 subscribe 进程内事件改为乐观更新 + mutation 后 refetch，MVP 不做实时推送
- **理由**: 单用户场景无需 WebSocket/SSE，简单够用
- **影响**: 前端需实现 mutation 后的数据刷新逻辑

### DEC-008: french-words 整合 MVP 范围

- **日期**: 2026-02-18
- **状态**: ✅ 已决策
- **提出者**: 小F（前端开发，研究分析）
- **决策者**: 小爽
- **决策**: MVP 只整合核心词汇追踪功能，暂缓扩展功能
