## 一、各模块规则设定

### 1. 能力画像 (Ability Profile)

#### 1.1 七维能力分类（系统固定）

系统预设 7 个能力大类，不可增删改：

| 维度 | 说明 |
|------|------|
| 技术能力 | 编程、架构等硬技能 |
| 产品思维 | 产品设计与用户理解 |
| 商业洞察 | 商业模式与市场判断 |
| 领导力 | 团队管理与影响力 |
| 沟通表达 | 口头/书面表达能力 |
| 创造力 | 创新与设计思维 |
| 学习能力 | 学习效率与方法论 |

#### 1.2 段位体系（Tier 0-5）

每个技能有一个当前段位（`currentTier`），取值 0-5：

| 段位 | 名称 | 含义 |
|------|------|------|
| T0 | 未开始 | 新建技能默认值 |
| T1 | 入门 | 1-3 个月练习 |
| T2 | 基础 | 6-12 个月练习 |
| T3 | 胜任 | 2-3 年系统训练 |
| T4 | 精通 | 专业水平或多年深耕 |
| T5 | 卓越 | 顶尖水平或获得业界认可 |

**升级规则**：
- **必须逐级完成**：完成 T3 里程碑前，必须先完成 T1 和 T2 的所有里程碑
- **自动升级**：当某段位的所有里程碑全部完成时，`currentTier` 自动 +1
- **上限为 T5**：到达 T5 后不再升级

#### 1.3 里程碑规则

每个里程碑有以下属性：

| 属性 | 可选值 | 说明 |
|------|--------|------|
| 类型 | `quantity`/`consistency`/`achievement`/`quality` | 里程碑的性质 |
| 验证方式 | `platform_auto`/`platform_test`/`evidence`/`self_declare` | 如何证明完成 |

**完成规则**：
- 每个里程碑只能完成一次（`milestoneCompletions` 有唯一约束）
- 完成时可提交证据 URL 和文字说明
- 已完成的里程碑不可撤销

#### 1.4 技能来源

技能创建有四种来源：`template`（模板库）、`ai`（AI 建议）、`custom`（手动创建）、`system`（系统预设）。

**AI 引导建档**：首次使用时 AI 对话帮助用户梳理技能，返回技能列表及预估段位，系统自动：
- 创建技能并归类到对应维度
- 从模板复制里程碑
- 将低于预估段位的里程碑标记为已完成（注明"自评初始建档"）

#### 1.5 徽章系统

| 徽章 | 触发条件 | 可撤回 |
|------|----------|--------|
| `first_milestone` | 完成任意 1 个里程碑 | 否 |
| `first_tier` | 任意技能解锁 T1 | 否 |
| `deep_mastery` | 任意技能达到 T5 | 否 |
| `lifelong_learner` | 3+ 活跃技能 | 是（删技能后标记 `historical`） |
| `polymath` | 5+ 不同能力大类有 T2+ 技能 | 是（删技能后标记 `historical`） |

**撤回机制**：删除技能时检查条件型徽章是否仍满足，不满足则标记为 `historical = true`。

#### 1.6 雷达图计算

```
每个维度分数 = avg(该维度下所有活跃技能的 currentTier × 20)
```

例如：技术能力维度有 3 个技能，段位分别为 T2、T3、T1，则分数 = avg(40, 60, 20) = 40 分（满分 100）。

#### 1.7 技能当前状态（Skill Current State）

- 每个技能可有多个状态键值对（如"工作语言: 法语"）
- 状态有过期天数（默认 180 天），超过后可能需要重新确认
- 通过 `(skillId, stateKey)` 唯一约束避免重复

#### 1.8 月度快照

- 按月保存雷达图数据和技能详情
- `date` 字段有唯一约束，每月最多一份
- 用于展示能力成长曲线

#### 1.9 活动日志（Activity Logs）

记录来自各模块的活动：

| source | 触发场景 | sourceRef 格式 | skillId |
|--------|----------|----------------|---------|
| `habit` | 习惯打卡 | `habitId` | 通过习惯关联的能力技能自动填充 |
| `planner` | 完成年度计划检查项 | `checkitem:{id}` | 通过目标关联的能力技能自动填充 |
| `milestone` | 完成里程碑/段位解锁 | 里程碑 ID | 里程碑所属技能 |
| `manual` | 用户手动添加 | — | — |
| `skill_learning` | 技能学习活动（关联了能力技能时） | — | 通过技能配置关联的能力技能自动填充 |
| `vocab` | 法语词汇 SRS 复习 | — | — |
| `article` | 收藏文章 | `article:{id}` | — |
| `project` | 完成项目 | `project:{id}` | — |

### 2. 日历打卡 (Habit Tracker)

#### 2.1 习惯频率

| 频率 | streak 计算 | 月完成率计算 |
|------|-------------|-------------|
| `daily` | 从今天/昨天往回数连续天数 | 已打卡天数 / 当月天数 |
| `weekly` | 连续有打卡的周数 | 有打卡的周数 / 当月总周数 |
| `monthly` | 连续有打卡的月数 | 有任何打卡则 100% |

#### 2.2 打卡规则

- **不可补未来**：打卡日期不能超过今天（服务端验证 `date <= today + 23:59:59`）
- **日期有效性**：会验证日期是否合法（如拒绝 2026-02-30）
- **切换机制（Toggle）**：同一天同一习惯再次打卡 = 取消打卡
- **乐观更新**：UI 立即反映变化，后台异步同步
- **习惯可归档**：`archived = true` 后不在活跃列表显示
- **关联能力技能**：习惯可通过 `linkedAbilitySkillId` 关联到能力画像中的技能，打卡时活动日志自动填充 `skillId` 和 `categoryId`

#### 2.3 连续天数（Streak）算法

```
1. 检查今天是否已打卡
2. 如果没打卡，从昨天开始计算
3. 往回逐天检查，遇到未打卡日期则中断
4. 返回连续天数
```

#### 2.4 ID 规则

习惯和打卡记录使用 UUID（唯一的例外，其他模块用自增整数）。

### 3. 年度计划 (Annual Planner)

#### 3.1 数据层级

```
年份(year) → 领域(domain) → 目标(goal) → 检查项(checkitem)
```

#### 3.2 优先级

目标有三级优先级：`high`（高）、`medium`（中）、`low`（低）。

#### 3.3 完成率计算

```
完成率 = 已完成检查项数 / 总检查项数 × 100%
```

按领域/按标签/总览三种维度统计。

#### 3.4 标签系统

- 标签名全局唯一
- 目标与标签是多对多关系
- 支持按标签视图聚合查看

#### 3.5 排序

领域、目标、检查项都有 `sortOrder` 字段，支持拖拽排序。

#### 3.6 跨年复制

可将上一年的领域和目标结构复制到新年份，检查项重置为未完成。

#### 3.7 检查项切换

- 勾选时记录 `completedAt` 时间戳
- 取消勾选时清除 `completedAt`
- **完成检查项时会向能力画像记录活动日志**，若所属目标关联了能力技能则自动填充 `skillId`

#### 3.8 关联能力技能

目标可通过 `linkedAbilitySkillId` 关联到能力画像中的技能，该目标下的所有检查项完成时，活动日志会自动带上对应的技能和维度信息。

### 4. 法语词汇 (Vocab Tracker)

#### 4.1 学习状态机

```
UNREAD ──→ TO_LEARN ──→ LEARNING ──→ MASTERED
              │            │             │
              └────────────┴─────────────┘
                    (可回退到 LEARNING)
```

**合法转换**：

| 动作 | 起始状态 | 目标状态 |
|------|----------|----------|
| SET_TO_LEARN | UNREAD | TO_LEARN |
| SET_LEARNING | UNREAD/TO_LEARN/MASTERED | LEARNING |
| SET_MASTERED | UNREAD/TO_LEARN/LEARNING | MASTERED |

**派生标记**：
- `isRead = true` 当 status ∈ {LEARNING, MASTERED}
- `isMastered = true` 当 status = MASTERED
- `firstInteractedAt` 在首次进入 LEARNING 或 MASTERED 时设置

#### 4.2 SRS 间隔重复算法（SM-2）

**核心常量**：

| 常量 | 值 | 说明 |
|------|----|------|
| NEW_WORDS_PER_SESSION | 20 | 每天最多学习新词数 |
| MAX_REVIEWS_PER_SESSION | 100 | 每天最多复习数 |
| AUTO_MASTERY_INTERVAL_DAYS | 30 | 间隔达此天数可标记掌握 |

**评分等级（0-5）**：

| 分数 | 含义 |
|------|------|
| 0 (AGAIN) | 完全忘记 |
| 2 (HARD) | 勉强记得 |
| 4 (GOOD) | 记住了 |
| 5 (EASY) | 非常简单 |

**算法核心**：

```
if quality < 3:
  repetitions = 0, interval = 1  // 重新开始
else:
  if repetitions == 0: interval = 1
  elif repetitions == 1: interval = 6
  else: interval = round(interval × easeFactor)
  repetitions += 1

easeFactor = max(1.3, easeFactor + (0.1 - (5-quality) × (0.08 + (5-quality) × 0.02)))
nextReviewAt = today + interval天（凌晨 00:00:00）
```

**easeFactor 初始值**：2.5

#### 4.3 每日学习计划

1. 查询到期复习词：`nextReviewAt <= now` 且 `repetitions > 0` 且非 MASTERED
2. 按 `nextReviewAt` 升序排序，上限 100 个
3. 查询今日已学新词数，如 < 20 则分配新词
4. 新词来源：LEARNING 状态但无 SRS 卡片的词汇，按 `rank` 排序

#### 4.4 释义缓存

AI 生成的词汇释义存入 `definitions` 表，永不失效（没有缓存淘汰策略）。

#### 4.5 多用户支持

词汇进度按 `userId` 隔离，支持多学习者独立进度。

### 5. 事项追踪 (Project Tracker)

#### 5.1 项目状态机

```
idea → todo → in_progress → done
  │     │        │
  │     │        ↓
  │     └──→ blocked ──→ done
  │                       │
  └──────→ dropped ←──────┘ (任何状态可进入 dropped)
```

实际实现中状态转换是**完全灵活的**（无强制约束），但 `blocked` 状态要求填写 `blockedReason`。

#### 5.2 优先级

项目和清单项都有优先级：`low`、`medium`、`high`。

#### 5.3 清单项规则

- 可设截止日期 `dueDate`
- 可关联里程碑 `milestoneId`
- 可关联笔记 `linkedNoteId`
- 支持完成/取消完成切换

#### 5.4 提醒系统

- 项目、里程碑、清单项都有 `reminderAt` 字段
- 通知表 `ptNotifications` 记录提醒是否已发送
- 系统定期检查到期提醒

#### 5.5 归档

- `archived = true` 实现软删除
- 归档后不在活跃列表中展示
- 与 `dropped` 状态独立（状态为 `dropped` 也可以不归档）

#### 5.6 笔记与附件

- 笔记支持 Markdown 格式
- 附件类型：`url`（链接）或 `image`（图片）
- 笔记可生成 AI 摘要

#### 5.7 图表

- 支持 Mermaid 等格式的项目图表
- 可存储图表类型、Mermaid 代码和描述

### 6. 文章阅读 (Article Reader)

#### 6.1 文章去重

按 URL 唯一约束去重，相同 URL 的文章不会重复导入。

#### 6.2 翻译缓存

每篇文章每种翻译类型（`full`/`summary`）最多一条缓存记录，再次翻译返回缓存结果。

#### 6.3 书签规则

每篇文章最多一个书签（`articleId` 唯一约束），书签可附带笔记。

#### 6.4 标签系统

- 标签名全局唯一，可设颜色
- 文章与标签多对多关系

#### 6.5 翻译流式输出

全文翻译使用 SSE（Server-Sent Events）流式返回，实时显示翻译进度。

### 7. 技能管理 (Skill Manager)

#### 7.1 技能配置

每个技能配置包含：
- 名称、描述、图标
- 知识树结构定义
- AI 提示词（教学/聊天/练习/测验各自独立）

#### 7.2 关联能力画像

技能配置可通过 `linkedAbilitySkillId` 关联到能力画像中的技能。关联后：
- 技能学习活动会自动写入能力画像的活动日志（`source='skill_learning'`）
- 活动日志自动带上对应的 `skillId` 和 `categoryId`
- 打通"学习工具"与"能力度量"的数据链路

#### 7.3 动态注册

- 创建的技能自动注册到侧边栏
- 删除技能配置后对应工具自动消失
- 支持 JSON 导入导出

#### 7.4 AI 知识树生成

可通过 AI 自动生成"领域 → 主题 → 知识点"的三级树状结构。

### 8. 技能学习引擎 (Skill Learning Core)

#### 8.1 知识点状态机

```
not_started → learning → understood → practiced
```

**特殊行为**：
- 调用 `generateTeaching()` 时自动将 `not_started` 改为 `learning`
- 进入 `understood` 或 `practiced` 时触发庆祝动画

#### 8.2 活动去重

同一类型 + 同一知识点在同一天内只记录一次（小时级去重）。

活动类型：`view`（查看）、`chat`（AI 对话）、`note`（笔记）、`task`（练习）、`status_change`（状态变更）、`quiz`（测验）。

#### 8.3 教学内容结构

AI 教学分五个 section：

| Section | 中文 | 说明 |
|---------|------|------|
| what | 是什么 | 概念定义 |
| how | 怎么做 | 方法论 |
| example | 例子 | 案例展示 |
| apply | 应用 | 实践场景 |
| resources | 资源 | 学习资料推荐 |

- 教学内容会缓存（按 `pointId` 唯一），再次访问直接返回
- 生成使用 SSE 流式输出

#### 8.4 测验系统

**题型**：
- `multiple_choice`：多选题
- `true_false`：判断题
- `fill_blank`：填空题
- `audio_choice`：听力选择题

**规则**：
- 每个答题产生一条记录（不限尝试次数）
- 判断正误基于精确/模糊匹配
- 教学内容生成后自动生成测验题

#### 8.5 推荐算法

推荐下一个知识点的逻辑：
1. 过滤非当前领域的未开始知识点
2. 优先推荐 `not_started`，其次 `learning`
3. 按 `sortOrder` 排序
4. 返回前 3 个

#### 8.6 连续学习天数

与习惯打卡类似的 streak 计算，基于 `smActivities` 表中的活动记录。

#### 8.7 文章关联

知识点可双向关联文章阅读器中的文章（`smPointArticles` 多对多表）。

### 9. 今日面板 (Dashboard)

#### 9.1 可领取里程碑提示

Dashboard 顶部展示可领取的里程碑。系统通过 `/api/ability-skills/claimable-milestones` 接口：
1. 查询所有未完成且 `verifyMethod = 'platform_auto'` 的里程碑
2. 利用 `verify.ts` 检查各模块数据是否满足里程碑条件
3. 满足条件的里程碑展示为可领取状态，用户点击"领取"即完成

#### 9.2 数据聚合

Dashboard 通过 `collectFullSummary()` 从所有模块拉取当日摘要：

| 数据源 | 展示内容 |
|--------|----------|
| 习惯打卡 | 活跃习惯数、今日完成数、最长连续天数 |
| 年度计划 | 领域数、目标数、检查项完成率 |
| 法语词汇 | 总词汇、已掌握、学习中、待复习 |
| 技能学习 | 各技能进度百分比 |
| 能力画像 | 技能列表与段位、雷达分数 |
| 文章阅读 | 总文章数、收藏数 |
| 事项追踪 | 活跃项目数、逾期事项数 |

#### 9.3 活动流

展示最近的 `activityLogs`，按 `createdAt` 倒序排列，可关联到具体技能。

### 10. 小爽助手 (Xiaoshuang AI Coach)

#### 10.1 上下文感知规则

根据用户消息中的关键词自动收集相关模块数据：

| 关键词模式 | 触发数据源 |
|-----------|-----------|
| 习惯/打卡/坚持/连续/streak/日常 | 习惯打卡 |
| 计划/目标/进度/完成率/年度 | 年度计划 |
| 法语/词汇/单词/复习/SRS/背单词/vocab | 法语词汇 |
| 学习/知识/课程/教程/skill learn | 技能学习 |
| 能力/技能/等级/段位/雷达/里程碑/成长/tier | 能力画像 |
| 文章/阅读/书签/article | 文章阅读 |
| 项目/事项/任务/project | 事项追踪 |
| 整体/全面/总结/回顾/情况/状态/所有 | 习惯+计划+词汇+学习+能力+文章+项目（全部 7 个模块） |

**无关键词匹配时**：默认收集「习惯」+「能力」数据。

#### 10.2 教练人设

- **严格诚实**：不恭维、不说空话
- **证据驱动**：基于实际数据给建议
- **可执行性**：每次回复必须有具体建议
- **跨模块洞察**：主动关联不同模块数据
- **反刷数据**：发现用户只是"刷数据"时直接指出

## 二、跨模块关系总览

```
┌─────────────────────────────────────────────────────────────────┐
│                        数据流向图                                │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│   ┌──────────┐    活动日志      ┌──────────────┐                │
│   │ 日历打卡  │ ──────────────→ │              │                │
│   └──────────┘  (打卡+技能绑定)  │              │                │
```

## 三、能力画像与其他模块的关系

### 3.1 直接数据写入（→ 能力画像）

| 来源模块 | 触发事件 | 写入内容 | skillId 来源 |
|----------|----------|----------|-------------|
| **日历打卡** | 每次打卡 | `source='habit'` | 习惯的 `linkedAbilitySkillId` |
| **年度计划** | 完成检查项 | `source='planner'` | 目标的 `linkedAbilitySkillId` |
| **法语词汇** | SRS 复习 | `source='vocab'` | — |
| **文章阅读** | 收藏文章 | `source='article'` | — |
| **事项追踪** | 完成项目 | `source='project'` | — |
| **技能学习** | 学习活动 | `source='skill_learning'` | 技能配置的 `linkedAbilitySkillId` |
| **能力画像自身** | 完成里程碑 | `source='milestone'` | 里程碑所属技能 |
| **能力画像自身** | 段位解锁 | `source='milestone'` | 同上 |
| **能力画像自身** | 创建技能 | `source='manual'` | 新建技能 ID |
| **能力画像自身** | 更新状态 | `source='manual'` | 对应技能 ID |

### 3.2 雷达图反映的跨模块积累

雷达图分数 = 每个维度下活跃技能段位的加权平均。段位的提升依赖于：

1. **里程碑完成**：直接提升技能段位 → 影响雷达图分数
2. **可领取里程碑**：`platform_auto` 类型的里程碑可通过 `/api/ability-skills/claimable-milestones` 自动检测是否满足条件，Dashboard 主动提示用户领取
3. **习惯打卡**（间接）：打卡记录进入活动日志（绑定技能后带 skillId）→ 反映用户在该技能方向的投入频率
4. **年度计划**（间接）：完成检查项进入活动日志（绑定技能后带 skillId）→ 同上
5. **技能学习**（间接）：关联了能力技能的学习活动写入活动日志 → 同上

### 3.3 Dashboard 读取能力画像

Dashboard 的摘要页面读取能力画像的：
- 所有活跃技能及其段位
- 雷达图分数
- 里程碑完成进度（已完成/总数）
- **可领取的里程碑**：自动检测 `platform_auto` 类型里程碑是否达标，展示提示卡片

### 3.4 小爽助手读取能力画像

当用户消息包含"能力/技能/段位/雷达"等关键词时，小爽收集：
- 所有技能列表（名称、大类、段位）
- 雷达分数
- 里程碑完成统计

小爽可以基于这些数据进行跨模块分析，例如：
- "你的打卡连续 30 天了，是否考虑提交里程碑？"
- "你的年度计划完成率较低，建议集中精力在核心目标上"
