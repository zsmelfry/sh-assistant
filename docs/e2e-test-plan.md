## 2. Phase 1 补充测试用例

### 2.1 补打卡（PRD 1.4 #6）

```
测试: 可对过去日期进行补打卡
步骤:
  1. 创建 daily 习惯
  2. 切换到上个月
  3. 点击上个月的第15天
  4. 验证该天显示 CheckedIn 标记
  5. 切换回当月，再回到上个月
  6. 验证第15天的打卡标记仍在
预期: 过去日期可正常打卡且持久化
```

### 2.2 跨浏览器数据共享（Phase 1 验收标准 #2）

```
测试: 不同浏览器上下文共享数据
步骤:
  1. 在 context A 创建习惯"跑步"并打卡第1天
  2. 打开新的 browser context B（模拟另一个浏览器）
  3. 在 context B 访问应用
  4. 验证 context B 中可以看到"跑步"习惯
  5. 验证 context B 中第1天有打卡标记
预期: 两个独立浏览器上下文看到相同数据
```

### 2.3 清除缓存后数据不丢失（Phase 1 验收标准 #3）

```
测试: 清除浏览器存储后数据仍在
步骤:
  1. 创建习惯并打卡
  2. 清除浏览器所有存储（localStorage, sessionStorage, IndexedDB, cookies）
  3. 刷新页面
  4. 验证习惯和打卡记录仍然存在
预期: 数据在服务端持久化，不依赖浏览器存储
```

### 2.4 API 响应时间（Phase 1 验收标准 #4）

```
测试: API 响应时间 < 100ms
步骤:
  1. 拦截所有 /api/* 请求
  2. 执行创建习惯、打卡、查询等操作
  3. 记录每个 API 请求的响应时间
  4. 验证所有响应时间 < 100ms
预期: 本地环境下 API 延迟在 100ms 以内
```

### 2.5 乐观更新体验（Phase 1 验收标准 #5）

```
测试: 打卡操作即时反馈（乐观更新）
步骤:
  1. 创建习惯
  2. 记录打卡点击时间
  3. 点击日期打卡
  4. 等待 UI 更新（CheckedIn class 出现）
  5. 计算时间差
  6. 验证 UI 更新时间 < 300ms
预期: UI 先于 API 响应更新
```

## 3. Phase 2 测试用例（图表功能）

### 3.1 历史面板基础交互

```
测试: 图表面板展开/折叠
步骤:
  1. 创建 daily 习惯
  2. 验证"查看历史"按钮可见
  3. 点击"查看历史"展开图表面板
  4. 验证热力图和趋势图区域可见
  5. 再次点击折叠
  6. 验证图表区域隐藏
预期: 面板默认折叠，可切换
```

### 3.2 Daily 习惯 - 年度热力图

```
测试: daily 习惯显示 7×52 热力图
步骤:
  1. 创建 daily 习惯并打卡若干天
  2. 展开历史面板
  3. 验证热力图容器可见
  4. 验证格子总数约为 364 (7行 × ~52列)
  5. 验证已打卡日期格子颜色为 #1A1A1A
  6. 验证未打卡日期格子颜色为 #EAEAEA
预期: 热力图正确渲染，颜色符合黑白规范

测试: 热力图 tooltip 交互
步骤:
  1. 创建 daily 习惯并打卡今天
  2. 展开历史面板
  3. 鼠标悬停在已打卡的格子上
  4. 验证 tooltip 显示日期和"已打卡"
  5. 鼠标悬停在未打卡的格子上
  6. 验证 tooltip 显示日期和"未打卡"
预期: tooltip 正确显示，延迟约 200ms

测试: 热力图年份导航
步骤:
  1. 创建习惯
  2. 展开历史面板
  3. 验证显示当前年份 (2026)
  4. 如果有上一年数据，点击"←"切换到 2025
  5. 验证年份标签更新
  6. 点击"→"切回 2026
预期: 年份导航正常，只能查看有数据的年份
```

### 3.3 Weekly 习惯 - 周完成格子视图

```
测试: weekly 习惯显示 1×52 周格子视图
步骤:
  1. 创建 weekly 习惯并打卡本周
  2. 展开历史面板
  3. 验证格子排列为一行（非 7×52 的日历格子）
  4. 验证约有 52 个格子
  5. 验证已完成周为 #1A1A1A，未完成周为 #EAEAEA
预期: 周视图正确渲染

测试: weekly 热力图 tooltip
步骤:
  1. 鼠标悬停在周格子上
  2. 验证 tooltip 显示"第N周 (M/D-M/D) 已完成/未完成"
预期: 周范围和状态正确显示
```
