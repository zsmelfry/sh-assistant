# 个人助手 — 产品功能书与使用手册

> 版本: v3.1 | 最后更新: 2026-03-24

## 产品概览

### 定位

面向个人用户的一站式成长管理平台，覆盖「目标规划 → 习惯养成 → 技能学习 → 知识积累 → 项目落地」的完整成长链路。

### 核心理念

- **数据驱动**：所有模块的进度、打卡、学习记录汇聚为统一数据，AI 教练可跨模块洞察
- **极简设计**：黑白单色视觉系统，CSS 变量控制主题，无视觉干扰
- **可扩展**：工具插件系统 + 技能学习引擎，新模块/新技能可低成本接入
- **本地优先**：SQLite 数据库本地部署，PM2 进程管理，隐私可控
- **多用户**：独立 Admin DB 管理用户，每用户独立数据库，模块级权限控制

### 技术栈

| 层级 | 技术 |
|------|------|
| 前端框架 | Nuxt 3 (SPA 模式) + Vue 3 Composition API |
| 状态管理 | Pinia (Composition API 风格) |
| 数据库 | SQLite (better-sqlite3) + Drizzle ORM，多用户隔离 |
| 认证 | JWT (365 天有效期) |
| AI | 插件式 LLM 接入 (Claude CLI / Claude API / Gemini / Ollama) |
| 部署 | PM2 进程管理 + 自动备份 |

## 系统架构概要

```
┌─────────────────────────────────────────────────┐
│                    前端 (SPA)                     │
│  ┌──────────┐  ┌──────────┐  ┌──────────┐       │
│  │  Pages    │  │  Stores  │  │Composable│       │
│  │ [...slug] │  │  (Pinia) │  │  (Auth,  │       │
│  │  → Tool   │  │  8 个    │  │  LLM..)  │       │
│  └──────────┘  └──────────┘  └──────────┘       │
│  ┌──────────────────────────────────────┐       │
│  │     Tool Plugin System (9 工具)       │       │
│  │  registerTool() → Map → [...slug]    │       │
│  └──────────────────────────────────────┘       │
│  ┌──────────────────────────────────────┐       │
│  │     小爽助手 (全局浮层)                │       │
│  └──────────────────────────────────────┘       │
├─────────────────────────────────────────────────┤
│                 Nitro Server                     │
│  Middleware: Security → Log → Auth → Guard      │
│  150+ API Routes (file-based routing)           │
│  LLM Providers (Claude CLI/API, Gemini, Ollama) │
│  Skill Learning Core (通用学习引擎)              │
├─────────────────────────────────────────────────┤
│            多用户 SQLite (WAL mode)              │
│  Admin DB (用户+权限) + User DBs (per-user)     │
│  13 Schema 文件 · Drizzle ORM · 自动迁移        │
└─────────────────────────────────────────────────┘
```

### 工具插件系统

每个工具模块是一个独立目录 (`tools/<id>/`)，包含：
- `index.ts` — 调用 `registerTool()` 注册工具元信息 (名称、图标、排序)
- 根组件 `.vue` — 延迟加载的工具界面
- `components/` — 工具内部子组件

路由自动解析：URL `/habit-tracker` → 查找 ID 为 `habit-tracker` 的工具 → 渲染对应组件。

## 模块详解

### 1. 今日 (Dashboard)

**入口**：侧边栏首项，应用启动默认页面

**功能概述**：
聚合所有模块的当日关键数据，提供一目了然的每日概览和 AI 驱动的个性化洞察。

**核心功能**：

| 功能 | 说明 |
|------|------|
| 每日问候 | 根据时间段显示问候语，附带当日摘要统计 |
| 可领取里程碑 | 自动检测已达标的里程碑，提示用户一键领取 |
| AI 洞察 | 基于全模块数据生成个性化每日建议 |
| 今日议程 | 展示当日待办目标和计划项 |
| 快速统计 | 打卡连续天数、学习进度、目标完成率等关键指标 |
| 近期动态 | 跨模块的最近活动流（含所有模块的活动日志） |

### 2. 能力画像 (Ability Profile)

**入口**：侧边栏「能力画像」

**功能概述**：
构建个人能力模型，通过 7 维雷达图可视化技能分布，使用里程碑分级体系追踪每项技能的成长路径。

**核心功能**：

| 功能 | 说明 |
|------|------|
| 7 维能力分类 | 技术能力、产品思维、商业洞察、领导力、沟通表达、创造力、学习能力 |
| 技能管理 | 添加技能（从模板或自定义），归类到能力维度 |
| 里程碑体系 | 每个技能 T0-T5 段位，由低到高逐级解锁 |
| 里程碑验证 | 提交证据完成里程碑，支持 AI 生成里程碑建议 |
| 雷达图 | 7 维 SVG 雷达图展示当前能力分布 |
| 成长曲线 | 月度快照记录能力变化趋势 |
| 荣誉墙 | 完成里程碑获得的成就徽章 |
| AI 引导 | 初次使用时 AI 对话帮助发现和定义技能 |

### 3. 法语词汇 (Vocab Tracker)

**入口**：侧边栏「法语词汇」

**功能概述**：
法语词汇学习工具，支持 CSV 批量导入，使用 SRS（间隔重复系统）科学记忆，LLM 生成词汇释义。

**核心功能**：

| 功能 | 说明 |
|------|------|
| CSV 导入 | 批量导入法语词汇列表 |
| 多用户 | 支持多个学习者独立进度 |
| 学习状态 | 未读 → 待学 → 学习中 → 已掌握 四阶段 |
| SRS 系统 | SM-2 间隔重复算法安排复习计划 |
| 每日计划 | 基于 SRS 算法推荐当日学习/复习词汇 |
| AI 释义 | LLM 生成词汇定义、例句（带缓存） |
| 学习模式 | 沉浸式学习界面 |
| 语音朗读 | Web Speech API 法语发音 |

| 领域分组 | 将目标按生活领域组织 |

| 检查项 | 目标拆解为可勾选的执行项 |
| 标签系统 | 跨目标标签分类，支持按标签视图聚合 |
| 年度复制 | 将上一年度结构复制到新年度 |

## 部署与运维

### 环境要求

- Node.js 18+
- SQLite 3
- PM2 (生产环境)
- Claude CLI (可选，AI 功能)
- Ollama (可选，AI 功能)

### 快速开始

```bash
# 1. 安装依赖
npm install

# 2. 配置环境变量
cp .env.example .env
# 编辑 .env 设置 JWT_SECRET 和 DATA_DIR

# 3. 初始化数据库
npm run db:migrate
npm run db:seed-user

# 4. 启动开发服务器
npm run dev
# 访问 http://localhost:3000
```

### 生产部署

```bash
# 一键部署（构建 + 备份 + 迁移 + PM2 重启）
npm run deploy
```

部署流程：
1. `npm run build` — 构建生产包到 `.output/`
2. 自动备份数据库到 `./data/backups/`（保留最近 5 份）
3. 迁移 Admin DB + 所有 User DB
4. PM2 重启服务进程

### 数据库管理

```bash
npm run db:generate  # 从 schema 变更生成迁移文件
npm run db:migrate   # 应用迁移
npm run db:studio    # 打开 Drizzle Studio 图形界面
```

- Admin DB：`data/admin.db`（用户 + 权限）
- User DB：`data/users/{username}.db`（各用户功能数据）
- WAL 模式，支持并发读取
- 外键约束已启用

### 日志与监控

```bash
npm run prod:logs    # 查看 PM2 日志
```

- 日志文件：`./logs/pm2-*.log`
- 请求日志：中间件自动记录所有 API 请求
