# 产品需求文档 — 小爽助手 v3.0

> 版本: v3.0
> 日期: 2026-03-08
> 状态: 待评审
> 前置版本: 能力画像系统 product-spec + PRD v2.0

---

## 一句话定义

**小爽助手**是一个以 AI 教练为中枢的个人成长操作系统——它记住你的一切、看见所有模块的数据、在你打开应用的瞬间就告诉你今天该做什么，并在你停滞时主动推你一把。

---

## 核心问题

当前系统有 7 个功能模块，但它们是**孤岛**：

| 问题 | 现状 | 影响 |
|------|------|------|
| AI 教练被困在能力画像里 | 只读 coach_profile（空）+ focus_plans（空）+ memories | 教练几乎没有上下文，回答空泛无用 |
| 模块间无联系 | 习惯、计划、技能学习、能力画像各自独立 | 用户无法感知"我今天的打卡和我的成长目标有什么关系" |
| 进入系统无动力 | 首页直接跳到能力画像工具 | 没有"今天该做什么"的引导感，冷启动 |
| 通知不够明显 | 通知只存在于能力画像页面内 | 其他模块的用户完全看不到提醒 |
| 里程碑可以跳级 | 没有前置 tier 校验 | 破坏了"逐级解锁"的成就感设计 |
| 荣誉墙与技能脱钩 | 技能增删不触发徽章更新 | 徽章状态可能过时 |

---

## 设计理念

### 小爽助手不是一个工具，是你的成长伙伴

1. **全局视角**：小爽看得到你所有模块的数据——打卡记录、计划进度、词汇学习、技能等级、阅读笔记。它不是从零开始的聊天机器人，而是一个了解你全部历史的长期教练。

2. **主动驱动**：你打开应用时，小爽已经准备好了今日议程。它不等你问"我今天该做什么"，而是直接告诉你。

3. **模块即手段**：习惯打卡、年度计划、技能学习——这些都是实现成长目标的手段。小爽帮你把它们串联成一条路径：目标 → 计划 → 习惯 → 每日行动。

4. **严格但温暖**：数据说话，不吹捧。但在你低落时给方向，在你突破时给认可。

## 核心功能

### 1. 小爽助手 — 全局 AI 教练

**从"能力画像内的聊天面板"升级为"系统级的 AI 伙伴"。**

#### 1.1 全局入口

小爽助手在任何页面都可以呼出：

- **桌面端**：侧边栏底部固定"小爽助手"按钮，点击展开右侧聊天面板（不离开当前页面）
- **移动端**：底部导航栏右下角浮动按钮，点击全屏打开聊天

聊天面板与当前工具共存，用户可以边操作边问小爽。

#### 1.2 全局数据感知

小爽在回答每个问题时，根据问题内容智能加载相关上下文：

| 数据源 | 加载内容 | 触发条件 |
|--------|---------|---------|
| 习惯打卡 | 活跃习惯列表、今日完成情况、各习惯连续天数、本月完成率 | 涉及习惯/坚持/日常的对话 |
| 年度计划 | 域/目标/检查项、完成率统计、逾期项 | 涉及目标/计划/进度的对话 |
| 技能学习 | 各技能知识树进度、学习活动热力图、最近学习内容 | 涉及学习/知识的对话 |
| 能力画像 | 技能等级、雷达图数据、焦点计划、里程碑进展 | 涉及能力/成长/评估的对话 |
| 法语词汇 | 总词汇量、SRS 待复习数、近期正确率 | 涉及法语/语言的对话 |
| 文章阅读 | 最近阅读文章、书签数量、标签分布 | 涉及阅读/文章的对话 |
| 事项追踪 | 活跃项目/里程碑、逾期事项 | 涉及项目/事项的对话 |
| 教练记忆 | 历史对话摘要（按相关度排序） | 所有对话 |

**上下文选择策略**：不是每次全部加载。根据用户消息的关键词和意图，选择 2-3 个最相关的数据源加载摘要。如果用户明确问"我的整体情况"，则加载全部数据源的概要。

#### 1.3 教练人设

```
你是小爽助手——用户的个人成长教练和助手。

## 你是谁
- 你了解用户的所有数据：习惯、计划、学习进度、能力等级
- 你记得之前的对话和用户的成长历程
- 你的目标是帮助用户实现自我认知与持续成长

## 你的风格
- 严格、诚实、以证据说话——不恭维、不说空话
- 但在用户低落时给方向，在用户突破时真诚认可
- 每次回复必须有具体、可执行的建议
- 用中文回复，语气自然亲切但不油腻

## 你的职责
- 帮用户梳理目标和优先级
- 基于数据分析成长模式（"你擅长短期冲刺但不擅长长期坚持"）
- 在停滞时调整策略
- 定期回顾整体画像，建议补短板
- 把不同模块的数据关联起来分析
```

#### 1.4 记忆系统

沿用现有 `coachMemories` 表，增强记忆管理：

- **自动记忆**：每次对话后提取关键洞察（现有机制保留）
- **记忆标签**：关联到具体 skillId / categoryId / 模块类型，提升检索相关度
- **记忆衰减**：超过 3 个月未被引用的记忆降低 importance
- **用户画像**：coach_profile.content 由小爽在周回顾时自动更新（不再依赖手动 PUT）

### 2. 仪表盘首页 — 每日动力源

**新增首页工具 `tools/dashboard/`，替代能力画像作为默认着陆页。**

#### 2.1 个性化问候

根据时间段和用户数据动态生成：

```
早上好，小爽！                          // 基于时间
🔥 你的法语词汇已经连续学习 47 天了       // 基于最长 streak
今天有 3 个习惯待打卡，2 个 SRS 词汇待复习  // 基于实际待办
```

#### 2.2 今日议程卡片

从所有模块自动聚合今日待办：

| 来源 | 内容示例 |
|------|---------|
| 习惯打卡 | ☐ 晨跑 ☐ 法语学习 ☑ 冥想 |
| SRS 复习 | 12 个词汇待复习 |
| 年度计划 | 2 个目标本周到期 |
| 技能学习 | 推荐学习：创业地图 → 第3章 |
| 焦点计划 | 法语：距离 B2 里程碑还差 2 个验证 |

每个条目可直接点击跳转到对应模块。

#### 2.3 小爽每日一言

基于实际数据生成的每日洞察/鼓励（缓存，每天最多生成一次）：

- 有进展时：「你这周完成了年度计划 3 个检查项，比上周多了 1 个。保持节奏。」
- 有风险时：「法语学习已经 5 天没动了，你的 B2 目标截止日还有 2 个月，建议今天至少复习 15 分钟。」
- 里程碑临近时：「你的词汇量还差 83 个就到 2000 了，冲一下？」

#### 2.4 快速数据概览

紧凑的数据摘要行：

```
[习惯] 今日 2/5 · 最长连续 47天  |  [词汇] 1917词 · 待复习12  |  [计划] 完成率 34%
```

#### 2.5 最近动态时间线

最近 5 条跨模块活动记录：

```
今天 09:15  完成习惯打卡「晨跑」
昨天 21:30  学习了 15 个新法语词汇
昨天 20:00  完成年度计划检查项「完成 React 课程」
3月5日      里程碑解锁：法语 → 基础 ★★
3月3日      获得徽章「万词王」
```

### 3. 全局通知系统

**通知从能力画像内部提升为系统级。**

#### 3.1 通知来源扩展

| 来源 | 通知类型 | 优先级 |
|------|---------|--------|
| 习惯打卡 | 连续记录即将断裂（晚8点仍未打卡） | high |
| 习惯打卡 | 连续记录创新高 | low（正向） |
| 年度计划 | 目标逾期未完成 | high |
| 年度计划 | 本周目标即将到期 | medium |
| 词汇学习 | SRS 待复习数 > 20 | medium |
| 技能学习 | 焦点技能 7 天未学习 | medium |
| 能力画像 | 状态即将过期 | low |
| 能力画像 | 里程碑达成 / 段位解锁 | low（正向） |
| 小爽助手 | 周回顾 / 月报告 | medium |

#### 3.2 全局通知展示

- **仪表盘首页**：重要通知以醒目横幅展示在顶部
- **侧边栏**：通知角标显示在"小爽助手"按钮上（而非单个工具）
- **聊天面板**：小爽主动提及待处理通知（"对了，你今天的晨跑还没打卡"）

#### 3.3 通知预算

- 每日最多 1 条 high + 3 条 medium/low
- 正向通知不计入预算
- 连续被忽略 3 次的通知类型自动降级

### 4. 里程碑逐级解锁

**修复：不允许跳级完成里程碑。**

#### 4.1 规则

- 完成 tier N 的里程碑前，必须先完成 tier N-1 的所有里程碑
- tier 1 无前置条件
- UI 上，未解锁 tier 的里程碑显示为锁定状态（灰色 + 锁图标）
- 点击锁定里程碑时，提示"需先完成 [前置tier名称] 的所有里程碑"

#### 4.2 校验位置

- **后端**：`POST /api/ability-skills/[skillId]/milestones/[milestoneId]/complete` 接口增加前置 tier 校验
- **前端**：渲染里程碑列表时标记 locked 状态，禁用完成按钮

### 5. 荣誉墙与技能联动

**修复：技能变更时同步更新徽章状态。**

#### 5.1 创建技能时

- 检查是否触发 `lifelong_learner`（3+ 活跃技能）
- 检查是否触发 `polymath`（5+ 大类各有 tier 2+ 技能）

#### 5.2 删除技能时

- 重新校验所有已授予的徽章是否仍满足条件
- 不满足条件的徽章：**不撤回**，但标记为"历史成就"（badge_awards 增加 `historical` 标志）
- 前端展示时区分"当前有效"和"历史成就"徽章

#### 5.3 展示关联

- 徽章详情页显示关联的技能
- 技能详情页显示该技能贡献解锁的徽章

### 6. 模块互联

**所有模块的数据不再是孤岛，通过两个机制互联。**

#### 6.1 统一活动日志

扩展现有 `activityLogs` 表，记录所有模块的关键活动：

| source | 记录内容 |
|--------|---------|
| habit | 打卡/取消打卡 |
| planner | 完成检查项、完成目标 |
| vocab | 学习新词、完成复习 |
| skill_learning | 完成知识点、完成练习 |
| ability | 里程碑完成、段位解锁 |
| article | 阅读完成、添加书签 |

这些日志供仪表盘时间线和小爽助手上下文使用。

#### 6.2 焦点计划联动

焦点计划已有 `linkedHabitIds`、`linkedPlannerGoalIds`、`linkedSkillLearningIds` 字段。增加：

- 小爽在创建焦点计划时自动建议关联哪些现有习惯/目标
- 仪表盘展示焦点计划时，直接显示关联模块的实时进度
- 焦点技能的关联习惯未打卡时，触发通知

## 用户旅程

### 场景1：早上打开应用

```
1. 用户登录 → 着陆仪表盘首页
2. 看到问候语 + 最长 streak 提醒
3. 看到今日议程：3个习惯待打卡、12个词汇待复习、1个计划检查项本周到期
4. 看到小爽每日一言："你的法语连续学习快到50天了，这周末冲一下？"
5. 直接点击"晨跑"跳转到习惯打卡，完成打卡
6. 回到仪表盘，看到"晨跑"已打勾
```

### 场景2：和小爽聊天

```
1. 用户在任意页面点击侧边栏"小爽助手"
2. 聊天面板滑出
3. 用户："我最近感觉没什么进步"
4. 小爽加载全局数据，回复：
   "从数据看，你这个月习惯打卡完成率 87%（比上月 92% 略降），
    法语词汇新增 45 个但复习正确率从 82% 降到 71%，
    年度计划只完成了 1 个检查项。
    我觉得核心问题是你在扩张（学新词）但没巩固（复习不够）。
    建议这周暂停新词学习，集中把待复习的 38 个词处理掉。"
5. 用户："好，帮我调整一下计划"
6. 小爽："建议把每日法语习惯从'学习+复习'改为纯复习模式，持续一周。
    要我帮你更新焦点计划吗？"
```

### 场景3：里程碑解锁

```
1. 用户完成法语基础段位最后一个里程碑
2. 系统校验：tier 1 所有里程碑已完成 → 解锁 tier 2
3. 弹出庆祝动效 + 显示新段位
4. 检查徽章：触发"first_tier"徽章授予
5. 仪表盘首页出现正向通知："恭喜！法语升级到基础 ★★"
6. 下次打开小爽时，它会主动提到这个成就
```

## 不做的事情

- **不做独立的通知中心页面**：通知通过仪表盘横幅 + 小爽聊天 + 侧边栏角标覆盖，不额外建页面
- **不做跨设备推送**：系统运行在 LAN 内，不接入 Push API，只在用户打开应用时展示
- **不做自动执行**：小爽只建议，不代替用户操作（不自动创建习惯、不自动修改计划）
- **不重写现有模块**：所有现有工具的功能保持不变，只在数据层增加活动日志接入

---

## 技术约束

- 沿用现有 Nuxt 3 + SQLite + Drizzle ORM 技术栈
- LLM 调用沿用现有 `resolveProvider` 机制（Claude CLI / Ollama）
- 全局数据采集只读，不修改其他模块的数据
- 仪表盘数据通过并行 API 调用聚合，不引入额外的聚合层
- 每日一言使用 LLM 生成，每天缓存一次，避免重复调用
