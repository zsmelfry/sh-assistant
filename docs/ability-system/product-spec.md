# 能力画像系统 — 产品功能书

## 一句话定义

一个以可量化技能为基础的自我认知与成长系统，通过里程碑解锁和能力验证追踪真实成长，让用户像看游戏角色面板一样了解自己，并在 AI 教练的持续指导下有针对性地提升。

## 设计理念

**不是为了取悦自己，而是为了真正认识自己。**

- 等级由完成的里程碑和通过的验证决定，不由活动时长或自我感觉决定。
- 里程碑必须包含质量验证——纯数量积累不能升级。
- 能力有"历史成就"和"当前状态"之分。当前状态会过期，需要定期确认。系统不美化你的现状。
- 只量化能量化的。不可客观衡量的维度不强行打分。
- 系统主动推动你，不是被动等你来。
- AI 教练严格、诚实、以证据说话。

## 目标用户

所有希望系统性提升自我的人。能力模型聚焦于可客观衡量的技能维度，适用于成年人和有家长指导的青少年。

## 核心概念

### 两层架构

```
第一层：技能成就（核心）
  用户拥有具体的技能，每个技能有里程碑体系。
  完成里程碑 + 通过验证 → 技能升级。

第二层：能力雷达（派生）
  每个技能归属一个能力大类。
  雷达图 = 各大类下技能的当前状态自动聚合。
  不是输入，是输出。
```

### 历史成就 vs 当前状态

每个技能同时维护两个维度：

**历史成就** — 你曾经做到过什么。永久记录，不可撤销。
> 2024年完成全马、2025年通过 DELF B2

**当前状态** — 你现在的水平。需要定期确认，会过期。
> 单次跑步能力：10公里（上次确认：2个月前）
> 法语词汇活跃度：每周复习 5 次（自动追踪）

雷达图基于当前状态计算。历史成就展示为荣誉墙和成长时间线。

---

## 能力大类

系统定义 7 个可量化的能力大类。只包含能通过客观指标衡量的维度：

| 大类 | 说明 | 典型技能示例 |
|------|------|-------------|
| 语言能力 | 母语及外语的听说读写 | 法语、英语、日语、写作 |
| 数理逻辑 | 数学、科学、技术、工程 | 编程、数学、数据分析、电子工程 |
| 身体运动 | 体能、运动项目、身体素质 | 跑步、游泳、力量训练、瑜伽 |
| 艺术创作 | 音乐、视觉、表演等创作技能 | 吉他、钢琴、绘画、摄影、视频制作 |
| 专业技能 | 职业相关的可验证专业能力 | 系统设计、产品管理、会计、投资 |
| 生活实践 | 日常生活中的实用技能 | 烹饪、驾驶、家居维修、急救 |
| 自我管理 | 通过行为数据可追踪的自律与习惯 | 习惯坚持度、目标完成率、学习持续性 |

**不包含的维度：** 情感心理、社交协作等不可客观量化的能力不纳入等级系统。AI 教练会在定期回顾中以对话方式讨论这些方面，但不赋予分数。

**"自我管理"的特殊性：** 这个大类不依赖自评，而是完全由平台行为数据驱动——习惯打卡的连续性、年度计划的完成率、学习活动的持续性。是唯一一个系统能全自动追踪的维度。

## 核心功能

### 1. 技能与里程碑

用户拥有一棵可扩展的技能树。每个技能下有分段位的里程碑体系。

**技能示例：法语**

```
法语 → 归属大类：语言能力
├── 入门 (★)
│   ├── 掌握 500 词 [平台自动 + 抽查验证]
│   ├── 完成基础语法模块 [平台记录]
│   └── 能读懂简单短文 [AI阅读测试]
├── 基础 (★★)
│   ├── 掌握 2000 词 [平台自动 + 抽查验证]
│   ├── SRS 复习正确率 > 75% [平台自动]
│   └── 能通过中级阅读理解测试 [AI测试]
├── 胜任 (★★★)
│   ├── 掌握 5000 词 [平台自动 + 抽查验证]
│   ├── 读完 10 篇法语长文并通过理解测试 [平台+AI]
│   └── 通过 DELF B1 或同等水平认证 [证据提交]
├── 精通 (★★★★)
│   ├── 掌握 10000 词 [平台自动 + 抽查验证]
│   ├── 读完 5 本法语原著 [证据提交]
│   └── 通过 DELF B2 或同等水平认证 [证据提交]
└── 卓越 (★★★★★)
    ├── 通过 DALF C1 或同等水平认证 [证据提交]
    └── 能用法语进行专业领域写作或演讲 [证据提交]
```

**技能示例：跑步**

```
跑步 → 归属大类：身体运动
├── 入门 (★)
│   ├── 单次完成 3 公里 [自证/运动app截图]
│   └── 连续 4 周每周跑步至少 1 次 [习惯打卡]
├── 基础 (★★)
│   ├── 单次完成 5 公里 [证据提交]
│   ├── 5公里配速 < 7分钟 [证据提交]
│   └── 连续 12 周每周跑步 [习惯打卡]
├── 胜任 (★★★)
│   ├── 单次完成 10 公里 [证据提交]
│   ├── 连续 6 个月每周跑步 [习惯打卡]
│   └── 累计跑量 500 公里 [记录追踪]
├── 精通 (★★★★)
│   ├── 完成半马 21 公里 [完赛证书]
│   └── 5公里配速 < 5分30秒 [证据提交]
└── 卓越 (★★★★★)
    ├── 完成全马 42 公里 [完赛证书]
    └── 半马成绩 < 1小时50分 [完赛证书]
```

**段位解锁规则：** 一个段位内的所有里程碑全部完成 + 通过验证 → 段位解锁，进入下一段位。

### 2. 里程碑验证机制

每个里程碑必须通过验证才算完成。进度系统以里程碑成就为主，在关键节点引入能力验证确认真实水平。

**四种验证方式：**

| 验证方式 | 说明 | 可信度标识 |
|----------|------|-----------|
| 平台自动 | 系统内数据自动判定（词汇量、打卡天数、学习进度） | 实心圆 |
| 平台验证 | 平台内触发测试/AI考核（词汇抽查、阅读理解测试） | 实心圆 |
| 证据提交 | 用户上传证书、截图、作品链接，记录在案 | 半实心圆 |
| 自评声明 | 用户声明达成，无外部证据（用于暂时无法验证的场景） | 空心圆 |

每个已完成里程碑旁显示对应的可信度标识。

**自评声明的定位：** 自评是务实的妥协方案——有些里程碑当前无法通过平台自动验证或 AI 考核（如"能在 10 人以上场合演讲"），允许用户先自评声明，标注为最低可信度。随着平台能力增强（如接入语音识别做口语测试），逐步用更客观的验证方式替代自评。系统明确标注哪些是自评、哪些是实测，不混淆。

**积累型里程碑的质量门槛：** 纯数量积累（如"掌握 5000 词"）不自动解锁。必须搭配质量验证——触发随机抽查（从已掌握词中抽 50 个测试，正确率 > 80%）或搭配其他质量型里程碑（如 SRS 正确率 > 75%）。这从机制上防止"刷数量"行为。

### 3. 能力雷达图

雷达图展示 7 个大类的综合水平，是技能当前状态的自动聚合结果，不需要任何手动评估。

**计算方式：**
- 每个技能的当前段位（1-5星）作为该技能的得分
- 大类得分 = 该大类下所有技能段位的平均值，归一化到 0-100
- 未到期的当前状态正常计入；已过期的降权显示

**数据充分度标识：**
- 大类下有 2 个以上活跃技能 → 实线显示，数据充分
- 大类下只有 1 个技能 → 虚线显示，标注"基于 1 项技能"
- 大类下无技能 → 不显示数值，灰色留空

雷达图不追求展示"完整的你"——它展示的是"你正在发展和追踪的领域"。空白不代表你不行，只代表你没在这里追踪。

### 4. 技能来源

技能从三个渠道进入系统：

**官方模板（最高质量）**
系统预置 20-30 个常见技能的完整里程碑体系，经过难度校准，验证方式明确。覆盖每个大类的典型技能。用户添加时一键获得，可微调。

**AI 生成（中等质量）**
用户描述想追踪的技能，AI 参考官方模板的难度锚点生成里程碑体系。用户确认或调整后生效。

**用户自定义（自由度最高）**
完全自由创建技能和里程碑。标注为"自定义"，不参与跨技能对比。

### 5. 焦点提升计划

人的精力有限。系统要求用户在任何时候只设定 1-3 个焦点技能，并为每个焦点生成可执行的提升方案。

一个焦点计划包含：
- **目标**：从当前段位到目标段位，截止日期
- **当前段位的未完成里程碑**：只展示这些，不展示更远的
- **AI 建议的训练策略**
- **关联行动**：直接链接到平台已有模块
  - 创建每日习惯 → 习惯打卡模块
  - 设定阶段目标 → 年度计划模块
  - 学习知识模块 → 技能学习模块
- **评估节点**：到达里程碑时触发验证

非焦点技能进入"维护模式"——只追踪当前状态，不推送进度，不催促。

### 6. AI 教练

系统内置一个持续了解用户的 AI 教练。它不是每次从零开始的聊天机器人，而是一个记得你所有历史、理解你当前状态的长期教练。

**教练的职责：**
- 帮你选择焦点技能（基于雷达图的薄弱区域和你的目标）
- 分析成长模式（"你擅长短期冲刺但不擅长长期坚持"）
- 在进度停滞时调整策略
- 定期回顾整体画像，建议补短板
- 在周/月回顾中讨论不可量化的方面（心理状态、社交关系等），但不打分
- 关注质量信号（"你这周背了 200 个新词，但复习正确率只有 60%，先巩固？"）

**教练如何"记住"你：**
教练维护一份持续更新的用户画像——你的技能状态、成长模式、性格特征、历史洞察。每次对话后自动提取关键信息更新画像，而非存储海量原始对话。对话时只加载画像摘要和相关历史片段，既保持深度理解又控制成本。

**画像的维护：**
- 画像中每条洞察带有记录时间
- 超过 3 个月未被新证据验证的洞察标记为"待确认"
- 用户可以随时查看和手动修正画像

### 7. 主动跟踪与提醒

系统不等用户主动打开，而是持续监控状态并主动推送。

| 频率 | 内容 |
|------|------|
| 每日 | 登录时显示今日焦点和下一个里程碑进度 |
| 每日 | 焦点关联习惯未完成时晚间提醒 |
| 每日 | 接近历史松懈点时预警（"你上次在第14天断了，明天是第14天"） |
| 每周 | 周回顾：本周活动汇总 + 焦点进展 + 下周建议 |
| 每月 | 月度报告：雷达快照对比 + 趋势分析 |
| 事件 | 里程碑达成 → 庆祝 + 展示下一个 |
| 事件 | 段位解锁 → 突出展示 + 徽章授予 |
| 事件 | streak 创新高 → 正向反馈 |
| 事件 | streak 断裂 → 不批评，问原因，帮助恢复 |
| 事件 | 焦点技能连续 7 天零活动 → 主动询问是否需要调整 |
| 事件 | 当前状态超过 6 个月未确认 → 提醒确认或标记过期 |

**通知预算：** 每天最多推送 1 条高优先级通知。多条时合并为摘要。用户可设安静模式。某类通知连续被忽略时自动降低频率。

### 8. 徽章系统

徽章是值得纪念的成就标记，从已完成的里程碑中自动产生。纯正向激励，不存在徽章被收回。

示例：
- **万词王** — 任意语言词汇量突破 10000
- **全马完赛者** — 完成 42 公里马拉松
- **年度坚持者** — 任意习惯连续打卡 365 天
- **全栈工匠** — 同时拥有前端和后端技能各达到胜任段位
- **博学者** — 在 5 个以上大类中各有至少 1 个技能达到基础段位
- **精深者** — 任意技能达到卓越段位

### 9. 成长记录

- **月度快照**：每月自动保存雷达图数据，用于历史对比
- **成长曲线**：按月/季度查看任意技能或大类的段位变化趋势
- **荣誉墙**：所有历史成就和徽章的时间线展示
- **活动日志**：所有与技能相关的活动记录（平台自动 + 手动记录平台外活动）
- **AI 报告**：月度/季度 AI 生成的成长分析

## 与现有模块的关系

能力画像不替代任何现有模块，而是在它们之上增加一层"为什么"和"通向哪里"：

```
能力画像（我要提升什么技能？下一个里程碑是什么？）
    │
    ├── 习惯打卡 → 自动反馈：连续性里程碑、自我管理大类数据
    ├── 年度计划 → 自动反馈：目标完成率、自我管理大类数据
    ├── 技能学习 → 自动反馈：知识点掌握度、对应技能里程碑
    ├── 词汇学习 → 自动反馈：词汇量、正确率、语言能力里程碑
    ├── 文章阅读 → 自动反馈：阅读量、对应技能里程碑
    └── 手动记录 → 平台外活动证据提交
```

## 初次使用流程

1. **AI 快速建档**（5 分钟）：AI 对话引导，了解用户已有技能和大致水平，批量初始化技能和当前状态。标记为"初始建档"验证方式。
2. **查看雷达图**：看到自己的初始画像，了解哪些大类有数据、哪些空白。
3. **选择焦点**（1-3 个技能）：AI 根据用户目标建议焦点，生成提升计划。
4. **开始执行**：完成里程碑、通过验证、解锁段位。系统持续跟踪和提醒。
5. **渐进丰满**：随着时间推移添加更多技能，雷达图逐渐丰满。不急，不强制。

## 设计边界

**这个系统是什么：**
- 一个可量化技能的追踪和成长系统
- 一个用客观里程碑衡量进步的工具
- 一个帮助你聚焦和坚持的 AI 教练

**这个系统不是什么：**
- 不是完整的人格画像工具（不量化情感、性格、社交能力）
- 不是精确的能力测评系统（里程碑和段位是粗粒度的）
- 不是与他人比较的排名系统（只和自己的过去比）
